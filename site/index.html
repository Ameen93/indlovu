How It Works

From ingestion to erasure — compliance at every step

Create Collection

Define dimensions, distance metric, and attach a POPIA or GDPR retention policy.

Insert Vectors

Add embeddings with metadata, source document tracking, and PII flags. Everything is audit-logged.

Search

k-NN vector search with metadata filtering. Results are logged but vectors are never leaked to audit.

Erase on Request

Right-to-erasure deletes all linked vectors across collections and generates a tamper-proof certificate.

Compliance Built In, Not Bolted On

Designed for POPIA and GDPR from the ground up

🇿🇦 POPIA

Section 24 — Right to have personal information deleted or destroyed. Indlovu provides cascading erasure with cryptographic proof that deletion occurred.


Section 19 — Retention must not be longer than necessary. Configurable retention policies auto-enforce time limits on PII-tagged collections.

🇪🇺 GDPR

Article 17 — Right to erasure ("right to be forgotten"). Full cascading deletion across all collections with audit-proof certificates.


Article 30 — Records of processing activities. Append-only audit logs track every operation on personal data.

Quick Start

Up and running in minutes

Rust
use indlovu_core::{Collection, Distance, VectorRecord};
use indlovu_compliance::{CompliantStore, InMemoryAuditLog, RetentionPolicy};

// Create a POPIA-compliant collection
let collection = Collection::new("customers", 384, Distance::Cosine)?;
let mut store = CompliantStore::new(collection, audit, "customers".into(), policy);

// Insert PII-tagged vectors with source tracking
let record = VectorRecord::new(vec![0.1; 384], metadata, Some("user-001".into()), true);
store.insert(record)?;

// Right-to-erasure — one call deletes everything
store.erase_by_source("user-001")?;
